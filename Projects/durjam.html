Development - August 2020

During the four day DurJam, my team of four set out to make a unique survival game. For all team members this was the first Collaborate project we had done and demonstrated why version control is so important. We initially used an online editor glitch which we wrote on at the same time. This allowed us to easily see others code as they were coding but a huge problem that was identified was not being able to test our code if anyone's code was incomplete. This was less of an issue during this short time frame as many of us did not code at the same time however it did lead us to copy a version to github to work simultaneously and merge our work at the end. We achieve some of our goals such as a randomly generated planet, rotating gravity at planet corners to allow player to walk all the way around the planet, collecting of resources, inventory system and functional survival bars such as oxygen and food. The player can then venture to a new planet getting in its ship with f and flying to a new planet. Objectives that still remained were adding specific ores to planets, adding more variety to planets, crafting system and loading resources to fuel the rocket.
